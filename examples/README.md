# Example outputs

Everything in this folder was generated by `npm run demo` against the bundled synthetic
Marketo instance (a fictional company, "Acme Corp", with four leads at different journey
stages). It is exactly what the tool will produce for your real instance — same files, same
schemas, real data.

| File | Produced by | What to look at |
|---|---|---|
| [`marketo-instance-map.md`](marketo-instance-map.md) | `mse recon` | The human-readable instance inventory: activity types, fields by category, forms, programs — and the **data-quality audit** at the bottom (suspect fields, dead campaigns, deprecated custom activities) |
| [`marketo-instance-map.json`](marketo-instance-map.json) | `mse recon` | Same content, machine-readable |
| [`signal-map.json`](signal-map.json) | `mse map` | The mapping contract. Note the `confidence` and `rationale` on every row — this is the file a human reviews and edits before anything downstream runs |
| [`snapshots/jane.doe@acme.com.md`](snapshots/jane.doe%40acme.com.md) | `mse snapshot` | The 9-section seller brief. Jane submitted "Contact Sales" after months of pricing visits and competitor research — this is what her rep sees 20 seconds later |
| [`snapshots/jane.doe@acme.com.json`](snapshots/jane.doe%40acme.com.json) | `mse snapshot` | The same brief plus the full journey blob (normalized events, stalls, committee) for programmatic use |
| [`signals.jsonl`](signals.jsonl) | `mse harvest --once` | Harvested state-change signals, one JSON object per line. Note `dedupeKey`, `strength`, `summary`, and `evidence` (raw Marketo activity IDs) on every signal |

Regenerate any time:

```bash
npm run demo && cp -R outputs/ examples/
```
