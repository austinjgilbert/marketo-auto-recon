# Support

## How to Get Help

### Bug Reports

If you've found a bug, please [open a GitHub issue](https://github.com/austinjgilbert/marketo-auto-recon/issues/new?template=bug_report.md)
using the bug report template. Include:

- Your Node.js version (`node --version`)
- The command you ran and the full error output
- A minimal reproduction if possible (redact any credentials)

### Feature Requests

Have an idea? [Open a feature request](https://github.com/austinjgilbert/marketo-auto-recon/issues/new?template=feature_request.md).
Describe the use case — what are you trying to accomplish and why does the current tool
fall short?

### Questions & Discussion

For general questions about usage, configuration, or Marketo API behavior, please use
[GitHub Discussions](https://github.com/austinjgilbert/marketo-auto-recon/discussions)
rather than opening an issue. This keeps the issue tracker focused on actionable bugs
and feature requests.

## Response Expectations

This is a solo-maintained open-source project. Response times are best-effort:

| Channel          | Typical response |
| ---------------- | ---------------- |
| Bug reports      | 1–2 weeks        |
| Feature requests | Reviewed monthly |
| Discussions      | Best-effort      |
| Security issues  | 48 hours (see [SECURITY.md](SECURITY.md)) |

## Security Issues

**Do not open public issues for security vulnerabilities.** See [SECURITY.md](SECURITY.md)
for the private disclosure process.

## Out of Scope

- Marketo API questions unrelated to this tool (consult [Marketo's developer docs](https://developers.marketo.com/))
- Anthropic/Claude API support (consult [Anthropic's documentation](https://docs.anthropic.com/))
- Custom integrations or consulting
